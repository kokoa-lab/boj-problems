## 문제

사이버국방학과 23학번인 민재(가명)는 오늘 해킹 기법 중 Return Oriented Programming(ROP)을 배웠다. ROP의 핵심은 공격자가 원하는 코드 내용을 이미 주어져 있는 바이너리 실행 파일의 내용으로 짜깁기해 만드는 것이다. 이 기법이 이해가 되지 않던 민재는 사이버국방학과를 졸업한 선배인 동우(가명)와 종우(가명)에게 ROP에 대해 물어봤다. 이에 동우는 다음과 같은 상황을 제시했다.

> 어떤 문자열 $S$가 주어져 있다고 생각해 봐. 너는 이 문자열 $S$의 부분 문자열들을 통해 또 다른 문자열 $T$를 만들고 싶어. 즉, $T$를 $T\_1T\_2\cdots T\_N$으로 나누는데, 각각의 $T\_i$는 $S$의 부분문자열이어야 해. 이미 주어져 있는 프로그램이 $S$, 너가 원하는 코드 내용이 $T$라고 생각하면 그게 ROP야. 여기서 $S$의 부분문자열이란, $S$의 앞과 뒤에서 $0$개 이상의 문자를 제거해 만들 수 있는 문자열을 의미해.

민재는 이를 듣고 다시 물어봤다.

> 그럼 만들 수 있는 방법이 여러 가지가 있다면, 어떤 방식이 좋을까요?

동우와 종우는 각각 다음과 같은 답을 내놓았다.

> 동우: $T$를 만드는 데 사용한 부분 문자열의 개수 $N$을 최소화해야 해.

> 종우: 아니지. $T$를 만들기 위해 사용한 부분 문자열의 길이를 모두 곱한 값인 $\prod\_{i=1}^N\lvert T\_i\rvert$를 최대화해야 해.

이를 본 민재는 선배들이 싸우지 않도록 다음 답을 모두 구하기로 했다.

1. $N$의 최솟값 $N\_{1}$과 이를 만족하는 경우가 여러 가지일 경우 $\prod\_{i=1}^N\lvert T\_i\rvert$의 최댓값 $P\_{1}$
2. $\prod\_{i=1}^N\lvert T\_i\rvert$의 최댓값 $P\_{2}$와 이를 만족하는 경우가 여러 가지일 경우 $N$의 최솟값 $N\_{2}$

단, $P\_1$과 $P\_2$는 수가 너무 커질 수 있으므로 답에 $\ln$을 취한 값을 구한다. 즉, 정답이 $P$라면 $e^{x}=P$를 만족하는 $x=\ln{P}$를 출력한다.

## 입력

첫 번째 줄에 알파벳 소문자로만 이루어진 문자열 $S(26\le\lvert S\rvert\le 100\, 000)$가 주어진다. $S$에는 `a`부터 `z`까지의 모든 알파벳 소문자가 각각 최소 한 개 이상 포함되어 있다.

두 번째 줄에 알파벳 소문자로만 이루어진 문자열 $T(1\le\lvert T\rvert\le 100\, 000)$가 주어진다.

## 출력

첫 번째 줄에 $N\_1$, $\ln{P\_1}$을 공백으로 구분하여 출력한다.

두 번째 줄에 $\ln{P\_2}$, $N\_2$를 공백으로 구분하여 출력한다.

주어진 조건을 만족하는 모든 $S$와 $T$에 대해 가능한 방법이 최소 한 가지 이상 존재한다.

정답과의 절대/상대 오차는 $10^{-6}$까지 허용한다.
